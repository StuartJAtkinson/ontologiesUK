Standing order ontology

A model describing standing orders for both Houses of Parliament, establishing persistent identification of orders and fragments of orders.

Classes

Business extent class #BusinessExtent

Sets out the extent of business covered by a standing order as public business or private business.

Fragment class #Fragment

A fragment is an FRBR-style work level item, wrapping different manifestations - described as fragment versions - over time. It represents a platonic entity and associated intent.

Fragment version class #FragmentVersion

A version of a fragment of a standing order, contained by a revision set.

House class #House

A parliamentary House, for example: the House of Lords.

Order class #Order

An order is an FRBR-style work level item, wrapping different manifestations - described as versions - over time. It represents a platonic entity and associated intent.

Order version class #OrderVersion

A version of an order contained by a revision set.

Revision class #Revision

A revision is a change to the text of a standing order fragment.

Revision set class #RevisionSet

A set of orders as they exist at the time of revision of one or more orders.

Suspension period class #SuspensionPeriod

A suspension period is the time during which one or more standing orders, or fragments of standing orders, are suspended. A suspension period has a start date and, where specified, an end date. These dates are set out in the utility model. Some suspensions do not specify an end date, for example: the suspension of House of Lords Standing Order 10(6) (Hereditary peers: by-elections) pending a further report from the Procedure and Privileges Committee. An order or fragment may be 'suspended, for a period' or 'dispensed with, for a purpose'. Orders and fragments may be dispensed with on a case by case basis: in the House of Commons this is usually indicated by the presence of 'notwithstanding' in the text of the motion. The suspension period class only covers suspensions and does not cover cases where standing orders are dispensed with. The procedure maps have steps which express the dispensing of standing orders.

Fragment version ordinality data property

Relates a fragment version to its position in the list of fragment versions contained by a revision set.

Domain: FragmentVersion. Range: integer.

Is major revision data property

Relates a revision to boolean describing if the revision is major: a FALSE value indicates only a change of case to the text content, a TRUE value indicates a major revision involving a change of wording to the text content.

Domain: Revision. Range: boolean.

Number data property

Relates a fragment version to its number as it appears, within a revision set.

Domain: FragmentVersion. Range: Literal.

Order version ordinality data property

Relates an order version to its position in the list of order versions contained by a revision set.

Domain: OrderVersion. Range: integer.

Revised on data property

Revision set ordinality data property
