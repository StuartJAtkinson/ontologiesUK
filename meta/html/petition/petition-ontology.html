Petition ontology

Petition ontology for UK Parliament. Covers the public surface area of petitions.

Classes

Approval class

Explicit moderation option for when a petition is approved.

ModerationOption (superclass) → Approval (class)

Debate class

A Parliamentary debate.

GovernmentResponse class

A response from the Government.

LocatedSignatureCount class

The number of signatures on a petition from a given place.

Moderation class

The moderation state of a petition. Might be awaiting moderation, rejected and unpublished, rejected and published or accepted and published.

ModerationOption class

The available moderation options for a petition. Might be awaiting moderation, rejected and unpublished, rejected and published or accepted and published.

Person class

A person.

Petition class

An online Parliamentary petition.

Place class

A place. Any kind of place.

Rejection class

Explicit moderation option for when a petition is rejected.

ModerationOption (superclass) → Rejection (class)

Threshold class

A type of threshold. Might be moderation threshold, government response threshold or debate threshold.

ThresholdAttainment class

The threshold attainment levels a petition might pass through from moderation threshold to government response threshold to debate threshold.

additionalDetails data property

Additional text to justify the petition entered by the petition creator.

background data property

A background explanation of why the petition was created entered by the petition creator.

Petition (domain) → background → langString (range)
