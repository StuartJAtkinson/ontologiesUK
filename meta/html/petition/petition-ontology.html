Ontologies

Petition ontology

Petition ontology for UK Parliament. Covers the public surface area of petitions.

Subclasses

petition: Approval
is a subclass of
petition: ModerationOption

petition: Rejection
is a subclass of
petition: ModerationOption

Classes

Approval class

Explicit moderation option for when a petition is approved.

ModerationOption (superclass) → Approval (class)

Debate class

A Parliamentary debate.

GovernmentResponse class

A response from the Government.

LocatedSignatureCount class

The number of signatures on a petition from a given place.

Moderation class

The moderation state of a petition. Might be awaiting moderation, rejected and unpublished, rejected and published or accepted and published.

ModerationOption class

The available moderation options for a petition. Might be awaiting moderation, rejected and unpublished, rejected and published or accepted and published.

Person class

A person.

Petition class

An online Parliamentary petition.

Place class

A place. Any kind of place.

Rejection class

Explicit moderation option for when a petition is rejected.

ModerationOption (superclass) → Rejection (class)

Threshold class

A type of threshold. Might be moderation threshold, government response threshold or debate threshold.

ThresholdAttainment class

The threshold attainment levels a petition might pass through from moderation threshold to government response threshold to debate threshold.

Data properties

action data property

The action text (title) of the petition entered by the petition creator.

Petition (domain) → action → langString (range)

additionalDetails data property

Additional text to justify the petition entered by the petition creator.

Petition (domain) → additionalDetails → langString (range)

background data property

A background explanation of why the petition was created entered by the petition creator.

Petition (domain) → background → langString (range)

closedAt data property

Date time at which the petition was closed.

Petition (domain) → closedAt → dateTime (range)

rejectionCode data property

A number identifying a specific rejection code.

Rejection (domain) → rejectionCode → integer (range)

rejectionDetails data property

Written explanation of why a petition was rejected.

Rejection (domain) → rejectionDetails → langString (range)

signatureCount data property

Number of signatures on a located signature count.

LocatedSignatureCount (domain) → signatureCount → integer (range)

signatureCountRetrievedAt data property

Date time at which a located signature count was retrieved from the petitions API.

LocatedSignatureCount (domain) → signatureCountRetrievedAt → dateTime (range)

thresholdAttainmentAt data property

Date time at which a threshold is met.

ThresholdAttainment (domain) → thresholdAttainmentAt → dateTime (range)

thresholdSignatureNumber data property

The number of signatures required to meet a threshold.

Threshold (domain) → thresholdSignatureNumber → integer (range)
