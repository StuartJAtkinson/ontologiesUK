Classes

Agent class

Agents include people and groups, including those outside the Parliamentary Service, participating in parliamentary procedure and contributing to the wider work of Parliament.

Archiving class

The action of archiving a work, indicating that no further revisions are intended to take place. Manifestations of the work which have not been withdrawn remain available, marked with an indication that no further updates are anticipated.

EditorialAction (superclass) → Archiving (class)

Editorial action class

An editorial action is an action taken on a manifestation or a work, for example: archiving a work, reviewing a manifestation, revising a manifestation or withdrawing a manifestation.

Editorial review class

A review of a manifestation. A decision to review may be triggered by an external event, for example: a change in Government policy, the availability of new data, external work in the same area or the output of parliamentary business; a change in user needs, for example: a new type of enquiry; or a change to library policy or practice, for example: in presentation, citation style, method of acknowledgement, change of portfolio or change of specialist.

EditorialAction (superclass) → Editorial review (class)

Feedback class

Feedback on a record, provided in a submission. An instance of the feedback class may contain the text of the feedback, or a pointer to the substance of the feedback in a separate system.

Manifestation class

A manifestation of an expression of a work, allowing for multiple versions of an expression over time, for example: an edition of a book, or a remastered audio file of a podcast episode.

Review cycle class

A review cycle describes the period between reviews of a work.

Revision class

The action of revising an existing manifestation, resulting in a new manifestation. The existing manifestation remains unchanged.

EditorialAction (superclass) → Revision (class)

Revision type class

The type of a revision, for example: a major or minor edit.

Withdrawal class

The action of withdrawing a manifestation. A manifestation may be withdrawn for a period and reinstated, or permanently withdrawn.

EditorialAction (superclass) → Withdrawal (class)

Work class

A distinct intellectual creation, similar to a work as defined in FRBR.

Object properties

Archiving of object property

Relates an archiving action to the work being archived.

Archiving (domain) → Archiving of (object property) → Work (range)

By object property

Relates an editorial action to the person or group performing that action.

EditorialAction (domain) → By (object property) → Agent (range)

Feedback leads to object property

Relates feedback to a resulting editorial review. An editorial review may be triggered by one or more items of feedback. The acceptance of feedback does not necessitate an editorial review. Feedback may result in more than one editorial review, or none.

Feedback (domain) → Feedback leads to (object property) → EditorialReview (range)

From manifestation object property

Relates an act of revision to an existing manifestation to be revised.

Revision (domain) → From manifestation (object property) → Manifestation (range)

Has review cycle object property

Relates a work to the review cycle of that work. A work has a single review cycle, or none.

Work (domain) → Has review cycle (object property) → ReviewCycle (range)

Has revision type object property

Relates an action of revision to the type of that revision.

Revision (domain) → Has revision type (object property) → RevisionType (range)

Leads to object property

Relates an editorial action to a resulting editorial action, for example: an editorial review leading to a revision of a manifestation.

EditorialAction (domain) → Leads to (object property) → EditorialAction (range)

Leads to new object property

Relates an editorial review to the creation of a new work.

EditorialReview (domain) → Leads to new (object property) → Work (range)

Makes object property

Relates an agent to the feedback submitted by that agent.

Agent (domain) → Makes (object property) → Feedback (range)

Results in object property

Relates a review cycle to resulting editorial reviews.

ReviewCycle (domain) → Results in (object property) → EditorialReview (range)

Review of object property

Relates an editorial review to the manifestation being reviewed.

EditorialReview (domain) → Review of (object property) → Manifestation (range)

Supersedes object property

Relates a work to a work that supersedes it: used to point from an archived work to its replacement. A work may be superseded by more than one work. A work may supersede more than one work.

Work (domain) → Supersedes (object property) → Work (range)

To manifestation object property

Relates an act of revision to a new manifestation resulting from the revision.

Revision (domain) → To manifestation (object property) → Manifestation (range)

Withdrawn by object property

Relates a manifestation to the action of withdrawing that manifestation.

Manifestation (domain) → Withdrawn by (object property) → Withdrawal (range)

Data properties

At data property

Relates an editorial action to the time at which the action was taken.

EditorialAction (domain) → At → dateTime (range)

Feedback text data property

Relates an item of feedback to the text of that feedback.

Feedback (domain) → Feedback text → langString (range)

Has revision description data property

Relates a revision - from an existing manifestation to a new manifestation - to a textual description of that revision. This is used to describe why the revision was made and is analogous to a git commit message.

Revision (domain) → Has revision description → langString (range)

Has revision type description data property

Relates a revision type to a textual description of that type.

RevisionType (domain) → Has revision type description → langString (range)

Published at data property

Relates a manifestation to the time at which it was published.

Manifestation (domain) → Published at → dateTime (range)

Received at data property

Relates an item of feedback to the time at which the feedback was received.

Feedback (domain) → Received at → dateTime (range)

Reinstated at data property

Relates a withdrawal of a manifestation to the time at which it was withdrawn.

Withdrawal (domain) → Reinstated at → dateTime (range)

Reviewed on data property

Relates an editorial review of a manifestation to the date on which the review took place.

EditorialReview (domain) → Reviewed on → date (range)
