AllocatedAnsweringBodyc back to ToC or Class ToC

IRI: http://parliament.uk/ontologies/written-question-and-answer/AllocatedAnsweringBody

The answering body allocated to a question by the Table Office or by an answering body transferring responsibility for a question. Note that the allocated answering body can change over time when one answering body transfers responsibility to another answering body. So allocated answering bodies are time bound. This would use startAt and endAt properties from the Core ontology.
is in domain of
allocatedToop
is in range of
allocationop, transferredFromop, transferredToop

AnswerTypec back to ToC or Class ToC

IRI: http://parliament.uk/ontologies/written-question-and-answer/AnswerType

The type of a given answer. e.g. holding answer, current answer or corrected answer. If a question is asked toward the end of a session and Parliament is prorogued before it can be answered, an answer of type prorogued may also be provided pointing out that there was not time to answer the question before prorogation.
is in range of
hasAnswerTypeop, previousAnswerop

WrittenQuestionc back to ToC or Class ToC

IRI: http://parliament.uk/ontologies/written-question-and-answer/WrittenQuestion

A type of Parliamentary question to which a written answer is expected. Written questions allow members to ask for information on the work, policy and activities of answering bodies. Usually this is to a minister in a government department but sometimes questions can be asked of non-departmental bodies. e.g. Church Commissioners.

allocationop, hasQuestionStatusop, headingop, inPursuantToop, namedDateForAnswerdp, questionTextop, refersToLibraryDepositop, tabledAtdp, templateTextop
is in range of

answerToop back to ToC or Object Property ToC

IRI: http://parliament.uk/ontologies/written-question-and-answer/answerTo

Allows a written answer to be associated with one or more written questions. This replicates in more detail the current functionality around grouped questions. In other words a group of questions in receipt of the same answer. There is some debate about whether the same textual answer to questions from multiple members can be considered as one answer. This also allows a single question to have many answers. Sometimes multiple answers from a single answering body (e.g. holding answer, current answer, correction). Sometimes different answers from different answering bodies to the same question. Note that this is a major change to current data models where a single question can have one and only one answer and a single answer can be to one and only one question.
has domain
WrittenAnswerc
has range
WrittenQuestionc
